\section{Problem Statement}

General LLMs can answer many questions but often lack domain or culturally specific Knowledge. 
The challenge we aim to address is building a system that can reliably and accurately answer complex, 
domain-specific questions about Portuguese cuisine and other recipes using structured data from the food.com dataset.

\section{Project Goal}

The goal of the project is to develop a ChatBot that combines a structured Knowledge Graph (KG) of recipes with Natural Language Processing (NLP) techniques and Retrieval-Augmented Generation (RAG).
The ChatBot will be able to interpret user queries in natural language, retrieve the most relevant recipes or recipe attributes, and provide accurate, context-aware responses.

\section{Data and Tools and Proposed Approach}

We will treat the dataset to include more of Portuguese cuisine recipes and keep the distribution of other cuisines. The structured Knowledge Graph (KG) is built using the RDF (Resource Description Framework) framework.
Each RDF triplet has the form:
\begin{itemize}
    \item \textbf{subject}: a specific recipe or ingredient (e.g., "Bacalhau à Brás"),
    \item \textbf{predicate}: an attribute such as \texttt{hasIngredient} or \texttt{hasPrepTime},
    \item \textbf{object}: the value of the attribute (e.g., "tomato", "15 minutes").
\end{itemize}
\noindent
The system first uses a fine-tuned BERT \cite{devlin2019bert} model to identify the user’s intent, such as finding a recipe and retrieving ingredients. Next, spaCy extracts relevant entities from the query, including ingredients, name of the recipe and cooking time, which are then linked to corresponding nodes or properties in the Knowledge Graph. The intent and linked entities are combined to generate a SPARQL query for precise retrieval of structured information. Each query has more context from the dataset, such as description, cooking steps and other tags (with information like diet, food restrictions etc) added with a RAG step to give Ollama more context through an embeddings-based retrieval that identifies the top-k most relevant recipes from textual fields, and a Retrieval-Augmented Generation (RAG). Finally, a React-based front-end presents the answers to the user in a conversational interface.
